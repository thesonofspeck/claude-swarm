# Claude Swarm

A native macOS app for running and managing many Claude Code sessions across
projects. Each task gets its own git worktree + branch, an embedded `claude`
CLI in a real terminal, plus integrated views of files, diffs, history, the
related Wrike task, and the resulting GitHub PR.

> Branded around the **Atom One** color scheme — light when your system is
> Light, dark when it's Dark.

## Quick start (Mac)

```sh
brew install xcodegen        # one-time
xcodegen generate            # creates ClaudeSwarm.xcodeproj
swift run --package-path Tools/AppIconGenerator render-icon  # one-time: builds the app icon PNGs
open ClaudeSwarm.xcodeproj
```

Xcode resolves the local Swift packages on first build. Press ⌘R.

## Workflow at a glance

1. **Add a project** — drop a folder onto the sidebar, or use ⌘⇧L. Map it to a
   Wrike folder if you want task-driven sessions.
2. **Pick a task** — Tasks tab lists open Wrike items.
3. **Start session** — opens a git worktree on a new branch, spawns `claude`
   as the **team-lead** agent, seeds the prompt with the task body, transitions
   the Wrike task to "In Progress."
4. **Watch & diff** — Files, Diff, History tabs auto-refresh as the agent
   edits. Switch to Terminal to interact directly. Transcript tab keeps the
   full scrollback even after the session ends.
5. **Open PR** — push & create via `gh`. Wrike auto-transitions to "In Review."
   Inline check-runs and review comments land in the PR tab; reply or resolve
   threads without leaving.
6. **Merge** — when GitHub reports `merged`, Wrike auto-transitions to "Done."

## Team library

Every teammate's Mac can pull a shared library of agents, skills, slash
commands, MCP servers, hooks, and a CLAUDE.md template from one place —
either a git repo or a local folder. Library detail tab inside any
project lets you install / uninstall / sync per item; the lock file
tells you when the team has updated something.

See [`docs/team-library.md`](docs/team-library.md) for the manifest
schema and a quick-start to bootstrap your team's library.

## Default agent team

Bootstrapped into every project's `.claude/agents/`:

- `team-lead` — orchestrator
- `ux-designer`, `systems-architect`, `engineer`, `qe`, `reviewer`

Plus a starter `CLAUDE.md` and a `.mcp.json` pointing at the bundled memory
server.

## Memory

Every session connects via stdio MCP to a bundled server that exposes:
`memory_write`, `memory_search`, `memory_get`, `memory_list`, `memory_delete`.
Namespaces: `global`, `project:<id>`, `session:<id>`. FTS5 search.

## Notifications

Claude Code's `Notification` and `Stop` hooks pipe into a Unix domain socket
the app listens on. Sessions waiting for input get a yellow pulse in the
sidebar, a dock-badge count, a macOS notification (suppressed when
foreground), and a count in the menu bar.

## iOS companion (Claude Swarm Remote)

Pair an iPhone to your Mac over local network or VPN to:

- See every Mac session and its status
- Get rich push notifications when an agent needs input or approval —
  Approve / Deny action buttons inline on the lock screen
- Tap a destructive tool call (`Bash(rm -rf …)`) and read the full
  prompt before approving
- Tap into a session and send a free-form message to the running `claude`

```sh
brew install xcodegen
cd ios && xcodegen generate
open ClaudeSwarmRemote.xcodeproj
```

The iOS app talks directly to the Mac on a WebSocket (port 7321 by default,
advertised over Bonjour as `_claudeswarm._tcp`). APNs pushes are sent
**directly from the Mac to Apple** using a `.p8` key uploaded in
**Settings → APNs** — no relay server.

When ≥1 paired iPhone is online, the Mac holds a
`PreventUserIdleSystemSleep` IOPMAssertion so hooks keep firing while
you step away. The assertion releases automatically on battery
(toggleable in Settings → iPhone).

See [`docs/ios-pairing.md`](docs/ios-pairing.md) for the full pairing +
APNs setup walkthrough.
